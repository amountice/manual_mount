📡 온라인 설치 가이드

사전 준비: 시스템 업데이트 및 기본 유틸리티 설치

가장 먼저 시스템을 최신 상태로 업데이트하고, 저장소 추가 등에 필요한 dnf-utils와 curl을 설치합니다.

sudo dnf update -y sudo dnf install -y dnf-utils curl

1단계: Git 설치

GitLab을 설치하기 전에 버전 관리 시스템인 Git을 먼저 설치합니다.

sudo dnf install -y git

2단계: Docker 설치 및 서비스 활성화

Docker 공식 저장소를 추가하고 Docker 관련 패키지들을 설치합니다.

# Docker 공식 저장소 추가 sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo # Docker 및 containerd 설치 sudo dnf install -y docker-ce docker-ce-cli containerd.io # Docker 서비스 시작 및 부팅 시 자동 실행 설정 sudo systemctl enable --now docker

3단계: GitLab CE (Community Edition) 설치 및 설정

GitLab은 리소스 사용량이 많으므로, 시스템 사양(최소 4 CPU core, 4GB RAM)을 확인하는 것이 좋습니다.

# GitLab 저장소 설정 스크립트 실행 curl -sS https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash # GitLab 설치 (EXTERNAL_URL은 실제 사용할 도메인이나 IP로 변경) sudo EXTERNAL_URL="http://gitlab.example.com" dnf install -y gitlab-ce # GitLab 초기 설정 및 실행 (시간이 다소 걸릴 수 있습니다) sudo gitlab-ctl reconfigure
⚠️ 중요: http://gitlab.example.com 부분은 실제 사용할 서버의 도메인 이름이나 IP 주소로 반드시 변경해야 합니다.

4단계: GitLab Runner 설치

GitLab CI/CD를 사용하려면 Runner를 설치해야 합니다.

# GitLab Runner 저장소 설정 스크립트 실행 curl -L "https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.rpm.sh" | sudo bash # GitLab Runner 설치 sudo dnf install -y gitlab-runner # GitLab Runner 서비스 시작 및 부팅 시 자동 실행 설정 sudo systemctl enable --now gitlab-runner
💡 참고: 설치 후, GitLab Runner를 웹 인터페이스에서 얻은 토큰을 사용하여 GitLab 인스턴스에 등록(register)해야 합니다.
(예: sudo gitlab-runner register)

5단계 (선택): Samba 및 NVIDIA Container Toolkit 설치

필요한 경우에만 설치합니다.

Samba (파일 공유)

sudo dnf install -y samba # Samba 서비스 활성화 sudo systemctl enable --now smb nmb

NVIDIA Container Toolkit (Docker에서 GPU 사용 시)

# NVIDIA 저장소 설정 curl -s -L https://nvidia.github.io/libnvidia-container/stable/rpm/nvidia-container-toolkit.repo | sudo tee /etc/yum.repos.d/nvidia-container-toolkit.repo # NVIDIA Container Toolkit 설치 sudo dnf install -y nvidia-container-toolkit # Docker가 NVIDIA 런타임을 사용하도록 재시작 sudo systemctl restart docker

💾 오프라인 설치 가이드

Part 1: 인터넷이 연결된 PC에서 준비 작업

먼저 인터넷이 연결된, 폐쇄망에 설치할 Rocky Linux와 동일한 버전/아키텍처의 PC에서 모든 파일을 다운로드해야 합니다.

1. RPM 파일들을 저장할 디렉토리 생성

mkdir ~/offline-rpms cd ~/offline-rpms

2. 필요한 외부 저장소(Repository) 추가

Docker와 GitLab은 기본 저장소에 없으므로, 해당 저장소 정보를 추가해야 dnf가 패키지를 찾을 수 있습니다.

# Docker 저장소 추가 sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo # GitLab CE 저장소 추가 curl -sS https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash # GitLab Runner 저장소 추가 curl -L "https://packages.gitlab.com/install/repositories/runner/gitlab-runner/script.rpm.sh" | sudo bash

3. 모든 필요 패키지 및 의존성 파일 다운로드

dnf download --resolve 명령어는 지정된 패키지뿐만 아니라, 설치에 필요한 모든 의존성 패키지까지 현재 디렉토리(~/offline-rpms)에 다운로드해 줍니다.

sudo dnf download --resolve \ git \ docker-ce \ docker-ce-cli \ containerd.io \ gitlab-ce \ gitlab-runner \ samba \ nvidia-container-toolkit
⚠️ 주의: 이 과정은 수백 개의 파일을 다운로드할 수 있으며 시간이 걸립니다.

4. USB로 파일 복사

다운로드가 완료되면 ~/offline-rpms 디렉토리 전체를 USB 저장소로 복사합니다.

# 예: /run/media/user/MyUSB 가 USB 마운트 경로일 경우 cp -r ~/offline-rpms /run/media/user/MyUSB/

Part 2: 폐쇄망 PC에서 설치 작업

이제 USB를 폐쇄망 PC에 연결하고 설치를 진행합니다.

1. USB에서 PC로 RPM 파일 복사

USB에 있는 offline-rpms 디렉토리를 폐쇄망 PC의 홈 디렉토리 등으로 복사합니다.

# 예: USB가 /dev/sdb1 이고 /mnt/usb에 마운트한 경우 mkdir ~/offline-rpms cp -r /mnt/usb/offline-rpms/* ~/offline-rpms/ cd ~/offline-rpms

2. 로컬 RPM 파일들로 설치 진행

dnf는 디렉토리 내의 모든 .rpm 파일을 대상으로 의존성을 분석하여 한 번에 설치할 수 있습니다.

# 현재 디렉토리의 모든 rpm 파일 설치 sudo dnf install ./*.rpm

이 명령을 실행하면 dnf가 offline-rpms 폴더 내의 파일들을 보고 필요한 순서대로 설치를 진행합니다.

3. 설치 후 설정 작업

온라인 설치와 마찬가지로, 설치가 완료된 후에는 각 서비스의 설정 및 활성화 작업이 필요합니다.

# Docker 서비스 활성화 sudo systemctl enable --now docker # GitLab 초기 설정 (시간이 오래 걸림) # http://gitlab.example.com 부분은 폐쇄망에서 사용할 IP나 호스트명으로 변경 sudo EXTERNAL_URL="http://your.closed.network.ip" gitlab-ctl reconfigure # GitLab Runner 서비스 활성화 sudo systemctl enable --now gitlab-runner # Samba 서비스 활성화 (필요 시) sudo systemctl enable --now smb nmb

폐쇄망 PC 오프라인 설치 전체 가이드

💡 사전 확인: dnf 명령어에 대하여
먼저, 가장 중요한 점입니다. Rocky Linux를 정상적으로 설치했다면 패키지 관리자인 dnf는 기본적으로 설치되어 있습니다. 따라서 dnf 명령어를 사용할 수 없는 걱정은 하지 않으셔도 됩니다. 이 가이드는 dnf를 사용하여 로컬 파일을 설치하는 방식입니다.

1단계: USB 장치 확인 및 마운트

PC에 USB를 꽂으면, 시스템이 USB를 인식하지만 바로 사용할 수는 없습니다. '마운트'라는 과정을 통해 시스템이 USB의 파일에 접근할 수 있도록 연결해 주어야 합니다.

1. USB 장치 이름 확인

USB를 꽂은 후, 아래 명령어를 실행하여 시스템에 연결된 디스크 목록을 확인합니다. sdb, sdc 등으로 표시되는 것이 USB일 확률이 높으며, 파티션 이름인 sdb1 또는 sdc1 등을 확인해야 합니다. (사이즈를 보고 판단하면 쉽습니다.)

lsblk

(예시 출력에서 `sdb1`이 USB인 것을 확인했다고 가정하겠습니다.)

2. 마운트 포인트 생성

USB를 연결할 비어있는 폴더를 하나 만듭니다.

sudo mkdir /mnt/usb

3. USB 마운트

확인한 장치 이름(sdb1)을 마운트 포인트(/mnt/usb)에 연결합니다.

# 아래 명령어에서 /dev/sdb1 부분은 1번 단계에서 확인한 실제 장치 이름으로 변경해야 합니다. sudo mount /dev/sdb1 /mnt/usb

2단계: 로컬 디스크로 파일 복사

USB에서 직접 설치하는 것보다, PC의 하드디스크로 파일을 모두 복사한 후 설치하는 것이 더 안정적입니다.

1. 홈 디렉토리로 파일 전체 복사

USB에 있는 offline-rpms 폴더를 현재 사용자의 홈 디렉토리(~/)로 복사합니다.

cp -r /mnt/usb/offline-rpms ~/

3단계: 프로그램 일괄 설치

이제 로컬에 복사된 RPM 파일들을 dnf를 사용하여 한 번에 설치합니다. dnf가 폴더 내의 모든 패키지를 분석하여 의존성 순서에 맞게 자동으로 설치해 줍니다.

1. RPM 파일이 있는 폴더로 이동

cd ~/offline-rpms

2. 전체 패키지 설치

현재 폴더에 있는 모든 .rpm 파일을 설치하라는 의미의 명령어입니다.

sudo dnf install ./*.rpm

(설치 과정에서 정말 설치할 것인지 묻는 `Is this ok [y/N]:` 질문이 나오면 `y`를 입력하고 Enter를 누르십시오.)

4단계: 설치 후 서비스 설정 및 활성화

소프트웨어 설치가 완료되었으니, 각 서비스를 시스템에 등록하고 실행하는 단계입니다.

1. Docker 서비스 시작 및 자동 실행 설정

sudo systemctl enable --now docker

2. GitLab 초기 설정

가장 중요한 단계입니다. EXTERNAL_URL에 폐쇄망에서 사용할 GitLab의 IP 주소나 호스트 이름을 정확하게 입력해야 합니다. 이 설정은 시간이 몇 분 이상 소요될 수 있습니다.

# http://192.168.1.100 부분을 실제 폐쇄망 PC의 IP 주소로 변경하세요. sudo EXTERNAL_URL="http://192.168.1.100" gitlab-ctl reconfigure

3. GitLab Runner 서비스 시작 및 자동 실행 설정

sudo systemctl enable --now gitlab-runner

(참고: GitLab Runner는 이 단계에서 서비스만 시작되며, GitLab에 연동(register)하는 작업은 GitLab UI에 접속하여 별도로 진행해야 합니다.)

5단계: USB 연결 해제 (마무리)

모든 작업이 끝났으면 USB를 안전하게 제거합니다.

sudo umount /mnt/usb

이제 USB를 물리적으로 뽑으시면 됩니다. 이 가이드의 4단계까지 완료되었다면, 지정한 IP 주소(예: http://192.168.1.100)로 웹 브라우저에서 접속하여 GitLab 초기 화면을 확인할 수 있습니다.

외장 SSD를 사용한 설치 방법

💡 시작 전: 모든 설치 파일이 들어있는 2TB 외장 SSD를 폐쇄망 PC에 연결합니다.

1단계: 외장 SSD 연결(마운트)

1. SSD 장치 이름 확인

아래 명령어로 디스크 목록을 확인하여 SSD의 장치 이름을 찾습니다. (예: /dev/sda1)

lsblk

2. 마운트 폴더 생성

SSD를 연결할 비어있는 폴더를 만듭니다.

sudo mkdir -p /mnt/ssd

3. SSD 마운트

위에서 확인한 장치 이름을 /mnt/ssd에 연결합니다.

# /dev/sda1 부분은 실제 확인한 장치 이름으로 변경하세요. sudo mount /dev/sda1 /mnt/ssd

2단계: 설치 파일들을 PC로 복사

PC의 홈 디렉토리로 offline-rpms 폴더를 전부 복사합니다.

cp -r /mnt/ssd/offline-rpms ~/

3단계: 모든 프로그램 일괄 설치

가장 중요한 단계입니다. 아래 명령어가 폴더 안의 모든 파일을 의존성에 맞게 자동으로 설치합니다.

1. 설치 폴더로 이동

cd ~/offline-rpms

2. 전체 패키지 설치

이 폴더에는 GitLab, Docker 뿐만 아니라, 이 PC가 exfat SSD를 읽는 데 필요한 exfatprogs와 fuse-exfat 드라이버도 함께 들어있습니다. dnf가 모든 것을 한 번에 설치해 줄 것입니다.

sudo dnf install ./*.rpm

설치 중 `Is this ok [y/N]:` 질문이 나오면 `y`를 입력하고 Enter를 누르세요.

4단계: 설치 후 서비스 설정

1. Docker 서비스 활성화

sudo systemctl enable --now docker

2. GitLab 초기 설정

EXTERNAL_URL에 폐쇄망에서 사용할 GitLab 접속 IP 주소를 입력합니다. 이 설정은 시간이 몇 분 이상 소요될 수 있습니다.

# http://your.closed.network.ip 부분을 실제 PC의 IP 주소로 변경하세요. sudo EXTERNAL_URL="http://your.closed.network.ip" gitlab-ctl reconfigure

3. GitLab Runner 서비스 활성화

sudo systemctl enable --now gitlab-runner

5단계: 마무리

모든 설치와 설정이 끝났습니다. SSD 연결을 해제합니다.

sudo umount /mnt/ssd

이제 웹 브라우저에서 설정한 IP 주소(예: http://your.closed.network.ip)로 접속하면 GitLab을 사용할 수 있습니다.

📦 설치 소프트웨어 목록

아래 목록은 폐쇄망 환경에 설치할 소프트웨어와 해당 소프트웨어를 다운로드하는 공식 출처를 명시한 것입니다. 모든 소프트웨어는 Rocky Linux의 공식 저장소 또는 각 개발사의 공식 패키지 저장소에서 제공됩니다.

1. Git (버전 관리 시스템)

  • 프로그램명: Git
  • 주요 패키지: git
  • 제공처: The Git Project
  • 출처 (저장소): Rocky Linux 공식 베이스 저장소 (AppStream)
  • 설명: 소스 코드 버전 관리를 위한 필수 프로그램이며, GitLab 운영의 기본 요소입니다.

2. Docker (컨테이너 플랫폼)

  • 프로그램명: Docker Community Edition (CE)
  • 주요 패키지: docker-ce, docker-ce-cli, containerd.io
  • 제공처: Docker, Inc.
  • 출처 (저장소): Docker 공식 저장소
  • 저장소 주소: https://download.docker.com/linux/centos/docker-ce.repo
  • 설명: 애플리케이션을 컨테이너 환경에서 실행하기 위한 플랫폼입니다. GitLab Runner가 CI/CD 작업을 컨테이너 내에서 수행할 때 사용됩니다.

3. GitLab (데브옵스 플랫폼)

  • 프로그램명: GitLab Community Edition (CE)
  • 주요 패키지: gitlab-ce
  • 제공처: GitLab, Inc.
  • 출처 (저장소): GitLab 공식 저장소
  • 저장소 주소: https://packages.gitlab.com/gitlab/gitlab-ce
  • 설명: 소스 코드 저장, CI/CD, 이슈 트래킹 등을 제공하는 올인원 데브옵스 플랫폼입니다.

4. GitLab Runner (CI/CD 실행기)

  • 프로그램명: GitLab Runner
  • 주요 패키지: gitlab-runner
  • 제공처: GitLab, Inc.
  • 출처 (저장소): GitLab Runner 공식 저장소
  • 저장소 주소: https://packages.gitlab.com/runner/gitlab-runner
  • 설명: GitLab CI/CD 파이프라인의 작업을 실제로 수행하는 에이전트입니다.

5. Samba (파일 공유 서비스) - 선택 사항

  • 프로그램명: Samba
  • 주요 패키지: samba
  • 제공처: The Samba Team
  • 출처 (저장소): Rocky Linux 공식 베이스 저장소 (AppStream)
  • 설명: Linux와 Windows 시스템 간 파일 및 프린터 공유를 위한 서비스입니다. (필요 시 설치)

6. NVIDIA Container Toolkit - 선택 사항

  • 프로그램명: NVIDIA Container Toolkit
  • 주요 패키지: nvidia-container-toolkit
  • 제공처: NVIDIA Corporation
  • 출처 (저장소): NVIDIA 공식 저장소
  • 저장소 주소: https://nvidia.github.io/libnvidia-container/stable/rpm/nvidia-container-toolkit.repo
  • 설명: Docker 컨테이너가 NVIDIA GPU 리소스를 사용할 수 있도록 지원하는 툴킷입니다. (GPU 활용 CI/CD 작업 시 필요)
